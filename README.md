# Podkop Unified Maintenance Tool

Монолитный интерактивный инструмент автоматизации, обновления и обслуживания экосистемы **Podkop**, ядра **sing-box** и сопутствующих компонентов на роутерах под управлением **OpenWrt** (совместим с версиями 22.03, 23.05 и новее, использующими файрвол `fw4` на базе `nftables`).

Скрипт полностью написан на чистом **POSIX sh**, не использует башизмы (вроде `local` в основном теле или массивы) и оптимизирован для работы в среде **BusyBox**.

---

## 🌟 Основные возможности

1. **Обновление ядра sing-box:** Автоматическая загрузка и замена расширенной версии бинарного файла от `EikeiDev/OpenWRT-sing-box-extended`.
2. **Обновление логики Podkop:** Подтягивание актуальных скриптов структуры из репозитория `irat25/podkop-auto-update`.
3. **Накатывание патчей в один клик:**
* Внедрение поддержки протоколов `xhttp` (`moix89/podkop-xhttp-patch`).
* Встроенный патч `wget -> curl` для исправления падений загрузки списков гео-IP при нестабильном соединении.


4. **Умная полная автоматизация (Full Auto-Update):** Ставит ночной апдейтер в `cron` и внедряет в него кастомный хук. Сразу после фонового обновления Podkop автоматически пересобираются и накатываются все патчи (`xhttp` и `curl`). Ваши модификации больше никогда не затрутся ночным обновлением.
5. **Идемпотентный Hard Kill Switch (Защита от утечки FakeIP):** Нативная блокировка утечки трафика в WAN при падении прокси (подробнее ниже).
6. **Правильный конвейер обслуживания (Full Maintenance):** Последовательное обновление всех слоев системы сверху вниз в один запуск: `Бинарь -> Структура -> Патчи -> Перезапуск -> Диагностика`.

## 🛡️ Архитектура Kill Switch (FakeIP Protection)

В отличие от устаревших решений, использующих нестабильные `init.d` скрипты с задержками `sleep` или привязку к `fwmark`, в данном инструменте Kill Switch реализован в соответствии с официальными гайдлайнами **OpenWrt fw4**:

* **UCI Integration:** Скрипт регистрирует конфигурационный блок `firewall.include` с типом `script`. Файрвол сам управляет жизненным циклом правил.
* **Абсолютная отказоустойчивость:** Правила применяются в момент старта подсистемы `fw4`, переживают любые команды `/etc/init.d/firewall restart` или изменения конфигурации интерфейсов в LuCI.
* **Идемпотентность:** Внутри инклуда реализованы строгие проверки через `nft list chain`. Правила **не дублируются** и не накапливаются при повторных вызовах.
* **Динамический анализ таблиц:** Перед взаимодействием с цепочкой `filter_forward` скрипт проверяет её фактическое существование, что исключает синтаксические ошибки на кастомных сборках OpenWrt.
* **Защита FakeIP подсети:** Блокировка срабатывает, если пакет с адресом назначения из пула `198.18.0.0/15` пытается уйти напрямую в интерфейс `wan`. Если `sing-box` упал — трафик уничтожается на месте.

## 🛑 Параноидальная защита от сбоев GitHub

Функция загрузки снабжена глубоким анализом текстовых ответов сервера. Скрипт перехватывает кастомные отказы GitHub API, блокировки, лимиты запросов и ошибки проксирования (`AccessDenied`, `Request blocked`, `abuse detection`, `Too Many Requests`, `rate limit exceeded`, `403`, `429`).

Если вместо bash-скрипта GitHub вернет HTML-страницу с ошибкой или заглушку блокировки, инструмент прервет операцию, не затерев рабочие системные файлы пустышками или битым кодом, и выведет предупреждение в консоль.

## 🚀 Быстрый запуск

Чтобы меню корректно отрабатывало нажатия цифр в терминале (без перехвата `stdin`), скачайте скрипт во временную директорию роутера и запустите его стандартным способом:

```sh
wget -qO /tmp/install.sh https://raw.githubusercontent.com/Vixald/podkop-tools/main/install.sh && sh /tmp/install.sh 

```

## 🛠️ Инструкция по использованию меню

При запуске скрипта перед вами откроется интерактивный терминальный интерфейс:

1. **Update sing-box** — Скачивает и обновляет только исполняемый файл прокси-клиента.
2. **Apply xHTTP & My curl patch** — Принудительно накатывает патчи поверх текущей развернутой версии Podkop.
3. **Setup Full Auto-Update** — Рекомендуемый пункт для первой настройки. Устанавливает расписание обновлений, прописывает хуки в `cron`, вычищает старые `init.d` скрипты киллсвича, удаляет мертвые симлинки из `/etc/rc.d/` и разворачивает нативный Kill Switch через UCI файрвола.
4. **Run Full Maintenance** — Запуск полного логического цикла обслуживания. Идеально подходит, если на роутере что-то сломалось или вы долго не обновляли компоненты.
5. **Restart Podkop** — Безопасный перезапуск службы с учетом структуры ваших путей установки (`/opt/etc/init.d/` или `/etc/init.d/`).
6. **Podkop global_check** — Встроенная утилита диагностики работоспособности туннелей и подсистем маршрутизации.
7. **0) Exit** — Корректный выход из утилиты с автоматической очисткой всех созданных временных файлов в `/tmp`.

## 📂 Структура разворачиваемых файлов

После активации автоматизации (Пункт 3) в системе создаются следующие компоненты:

* `/usr/share/podkop_killswitch.nft` — Системный скрипт правил файрвола с условными проверками `nftables`.
* `/root/podkop-patch-hook.sh` — Скрипт-обработчик, который вызывается планировщиком `cron` сразу после выполнения основного автоапдейтера для возврата патчей на место.
